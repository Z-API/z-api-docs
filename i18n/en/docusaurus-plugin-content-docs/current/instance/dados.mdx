# Get Instance Data

The instance data retrieval endpoint is one of the most fundamental operations in the Z-API API. It allows you to retrieve complete and up-to-date information about your instance, including the current connection status with WhatsApp, all active settings, technical details of the connected device, and webhook configurations.

**Why is this endpoint important?**

This endpoint is essential for monitoring and diagnostics. Before sending messages, you should verify if the instance is connected (`status: "connected"`). It also provides valuable information about the connected WhatsApp device (app version, operating system, device model), which can be useful for troubleshooting and understanding the execution environment.

**When to use this endpoint:**

- **Status Check**: Before sending messages, verify if the instance is connected
- **Monitoring**: Implement periodic checks to monitor the health of the instance
- **Diagnosis**: Use device information to diagnose compatibility issues
- **Configuration**: Verify current webhook settings and other options

## Endpoint

```http
GET https://api.z-api.io/instances/{instanceId}
```

### Headers

- Client-Token: string (required)

## Responses

### 200 OK

```json
{
 "id": "3C3F8E5F4A2B1C9D",
 "name": "Minha Instância",
 "status": "connected",
 "qrcode": null,
 "phone": {
 "wa_version": "2.24.10.79",
 "mcc": "724",
 "mnc": "05",
 "os_version": "Android 13",
 "device_manufacturer": "Samsung",
 "device_model": "SM-G991B"
 },
 "webhook": {
 "url": "https://seusite.com/webhook",
 "events": ["message", "status"]
 }
}
```

### Response Fields

| Field | Type | Description |
|------------|--------|----------------------------------------------|
| id | string | Unique identifier of the instance |
| name | string | Instance name |
| status | string | Current status of the instance (`connected`, `disconnected`, `qrcode`) |
| qrcode | string | QR Code (if available, otherwise `null`) |
| phone | object | Connected device information |
| webhook | object | Webhook configurations |

### Common Errors

| Code | Reason | How to Resolve |
|--------|-----------------------|---------------------------------------|
| 401 | Invalid token | Check the header `Client-Token` |
| 404 | Instance does not exist | Confirm the `instanceId` |
| 5xx | Internal error | Try again; open support if it persists |

```javascript
const https = require('https');

// ⚠️ SEGURANÇA: Use variáveis de ambiente para credenciais
const instanceId = process.env.ZAPI_INSTANCE_ID || 'SUA_INSTANCIA';
const clientToken = process.env.ZAPI_CLIENT_TOKEN || 'seu-token-de-seguranca';

// Obter dados da instância
function getInstanceData() {
  return new Promise((resolve, reject) => {
    const path = `/instances/${encodeURIComponent(instanceId)}`;
    
    const options = {
      hostname: 'api.z-api.io',
      path: path,
      method: 'GET',
      headers: {
        'Client-Token': clientToken,
      },
    };

    const req = https.request(options, (res) => {
      let data = '';

      res.on('data', (chunk) => {
        data += chunk;
      });

      res.on('end', () => {
        if (res.statusCode >= 200 && res.statusCode < 300) {
          try {
            const result = JSON.parse(data);
            // ⚠️ SEGURANÇA: Não logue tokens ou dados sensíveis
            console.log('Dados da instância obtidos com sucesso');
            console.log('ID:', result.id);
            console.log('Nome:', result.name);
            resolve(result);
          } catch (error) {
            reject(new Error('Erro ao parsear resposta JSON'));
          }
        } else {
          // ⚠️ SEGURANÇA: Não exponha detalhes sensíveis em logs
          reject(new Error(`Erro HTTP ${res.statusCode}`));
        }
      });
    });

    req.on('error', (error) => {
      console.error('Erro na requisição:', error.message);
      reject(error);
    });

    req.end();
  });
}

// Executar
getInstanceData()
```

```java
public class GetInstanceData {
    // ⚠️ SEGURANÇA: Use variáveis de ambiente para credenciais
    private static final String INSTANCE_ID = System.getenv("ZAPI_INSTANCE_ID") != null 
        ? System.getenv("ZAPI_INSTANCE_ID") : "SUA_INSTANCIA";

            // ⚠️ SEGURANÇA: Sempre use HTTPS
            String urlString = String.format(
                "https://api.z-api.io/instances/%s",
                java.net.URLEncoder.encode(INSTANCE_ID, StandardCharsets.UTF_8)
            );
            
            URL url = new URL(urlString);
            HttpURLConnection connection = (HttpURLConnection) url.openConnection();
            connection.setRequestMethod("GET");
            connection.setRequestProperty("Client-Token", CLIENT_TOKEN);
            connection.setConnectTimeout(30000);
            connection.setReadTimeout(30000);

            int responseCode = connection.getResponseCode();
            
            if (responseCode >= 200 && responseCode < 300) {
                BufferedReader reader = new BufferedReader(
                    new InputStreamReader(connection.getInputStream(), StandardCharsets.UTF_8)
                );
                StringBuilder response = new StringBuilder();
                String line;
                while ((line = reader.readLine()) != null) {
                    response.append(line);
                }
                reader.close();
                
                // ⚠️ SEGURANÇA: Não logue tokens ou dados sensíveis
                System.out.println("Dados da instância obtidos com sucesso");
                System.out.println(response.toString());

                // ⚠️ SEGURANÇA: Não exponha detalhes sensíveis em logs

            }

        } catch (Exception e) {

        }
    }
}
```

```go
package main

import (

    "net/http"

func main() {
    // ⚠️ SEGURANÇA: Use variáveis de ambiente para credenciais
    instanceId := os.Getenv("ZAPI_INSTANCE_ID")
    if instanceId == "" {
        instanceId = "SUA_INSTANCIA"
    }
    
    clientToken := os.Getenv("ZAPI_CLIENT_TOKEN")
    if clientToken == "" {
        clientToken = "seu-token-de-seguranca"
    }

    // ⚠️ SEGURANÇA: Sempre use HTTPS
    url := fmt.Sprintf("https://api.z-api.io/instances/%s", instanceId)
    
    client := &http.Client{
        Timeout: 30 * time.Second,
    }
    
    req, err := http.NewRequest("GET", url, nil)
    if err != nil {
        fmt.Printf("Erro ao criar requisição: %v\n", err)
        return
    }
    
    req.Header.Set("Client-Token", clientToken)
    
    resp, err := client.Do(req)
    if err != nil {
        fmt.Printf("Erro na requisição: %v\n", err)
        return
    }
    defer resp.Body.Close()
    
    if resp.StatusCode >= 200 && resp.StatusCode < 300 {
        body, err := io.ReadAll(resp.Body)
        if err != nil {
            fmt.Printf("Erro ao ler resposta: %v\n", err)
            return
        }
        
        var result map[string]interface{}
        if err := json.Unmarshal(body, &result); err != nil {
            fmt.Printf("Erro ao parsear JSON: %v\n", err)
            return
        }
        
        // ⚠️ SEGURANÇA: Não logue tokens ou dados sensíveis
        fmt.Println("Dados da instância obtidos com sucesso")
        fmt.Printf("ID: %v\n", result["id"])

    } else {
        // ⚠️ SEGURANÇA: Não exponha detalhes sensíveis em logs
```

## Understanding the Returned Data

### Instance Status

The field `status` can have three main values:

- **`"connected"`**: The instance is connected and ready to send and receive messages. This is the ideal state for normal operation.
- **`"disconnected"`**: The instance is disconnected from WhatsApp. You will need to reconnect by scanning a new QR Code.
- **`"qrcode"`**: The instance is waiting for authentication. A QR Code is available in the field `qrcode` to be scanned.

### Authentication QR Code

The field `qrcode` contains a base64 encoded string with the QR code image when the instance is disconnected or waiting for authentication. When the instance is already connected, this field returns `null`. 

**How to use the QR Code:**
1. Decode the base64 string to get the image
2. Display the image for the user to scan with WhatsApp
3. After scanning, the status will change to `"connected"`

### Device Information

The object `phone` contains valuable technical information about the connected WhatsApp device:

- **`wa_version`**: Version of WhatsApp installed on the device. Useful for verifying compatibility with specific features.
- **`os_version`**: Operating system and version (e.g., "Android 13", "iOS 17.0"). Helps in diagnosing platform-specific issues.
- **`device_manufacturer`** and **`device_model`**: Manufacturer and model of the device. Useful for technical support and troubleshooting.

**Important**: These information are automatically updated when the instance connects to WhatsApp. If you do not see these details, the instance may not be fully connected.

### Webhook Configurations

The object `webhook` displays all active webhook configurations of your instance:

- **`url`**: The URL configured to receive real-time events
- **`events`**: Array with the types of events configured (e.g., `["message", "status"]`)

Use these information to verify if your webhooks are correctly configured and which events are being monitored.

## Practical Use Cases

### Status Check Before Sending Messages

Always check the instance status before attempting to send messages to avoid errors:

```javascript
const instanceData = await getInstanceData();
if (instanceData.status !== 'connected') {
  console.error('Instância não está conectada. Status:', instanceData.status);
  // Lógica para reconectar ou notificar o usuário
}
```

### Automatic Monitoring

Implement periodic checks to monitor the health of the instance:

```javascript
// Verificar status a cada 5 minutos
setInterval(async () => {
  const data = await getInstanceData();
  if (data.status === 'disconnected') {
    // Notificar administrador ou tentar reconectar
    notifyAdmin('Instância desconectada!');
  }
}, 5 * 60 * 1000);
```

### Troubleshooting

Use device information to diagnose issues:

```javascript
const data = await getInstanceData();
if (data.phone) {
```
